# Router training data

`router_questions.jsonl` тАФ labelled questions for the router classifier (`{"question", "label"}`,
one JSON object per line). 255 questions, balanced 85 / 85 / 85 across `factual`,
`comparative`, `multihop`.

## How it's made

Generated by `scripts/generate_router_data.py` (fixed seed, reproducible). Labels are reliable
**by construction** тАФ a factual template only ever emits a factual question тАФ so there is no
pile of unlabelled questions to hand-annotate.

Each category mixes several phrasings with a block that deliberately drops the obvious keyword
(a comparative without "compare", a multi-hop without "and how long"), so the classifier learns
intent rather than a surface cue. Vocabulary is mostly insurance/healthcare (matching the app's
own examples) plus some general-knowledge questions.

## Known limitations (read before citing accuracy)

- **Synthetic, not real user queries.** Accuracy on a held-out split of this set measures how well
  the classifier separates these templates тАФ an optimistic proxy for messy real questions.
- **Premises can be nonsensical** (e.g. "What year did Alan Turing publish Inception?"). The label
  is still correct: the router classifies question *type*, not whether the question is answerable.
- **A human spot-check is still worth doing** before you report the number as a headline metric.
  The construction makes labels trustworthy, but a few minutes eyeballing a sample catches template
  bugs the seed would otherwise bake in.
